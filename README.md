# UDP Server Performance Iteration

A step-by-step progression of UDP echo server implementations in C, each removing one specific bottleneck. The goal is to demonstrate measurable performance improvement at every layer ‚Äî from a naive blocking server to kernel-bypass via AF_XDP.

| Version | Name             | Implementation & Benefit                                                                                                                                                                                                                                   | Bottleneck / Problem                                                                                     |
| ------- | ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| `v1`    | `blocking_st`    | Single thread, `recvfrom`/`sendto` loop. Simplest possible baseline                                                                                                                                                                                        | Single core; one syscall per packet; thread stalls inside kernel ‚Äî CPU idles                             |
| `v2`    | `blocking_mt`    | N threads at startup + `SO_REUSEPORT`, each runs the v1 loop independently. Kernel distributes packets across sockets. Throughput scales with core count                                                                                                   | Each thread stalls on `recvfrom`; OS thread overhead (~1MB stack, context switches)                      |
| `v3`    | `nonblocking_st` | Single thread, `epoll` + `O_NONBLOCK` socket. Thread never enters kernel sleep ‚Äî returns immediately if no packet ready                                                                                                                                    | Single core ceiling; `epoll` overhead wasted on UDP (socket nearly always ready); one syscall per packet |
| `v4`    | `nonblocking_mt` | N threads + `SO_REUSEPORT`, each owns a private `epoll` instance. No shared state ‚Äî tight independent event loop per thread                                                                                                                                | One syscall per packet per thread ‚Äî kernel/userspace transition dominates at high pps                    |
| `v5`    | `mmsg_mt`        | v4 loop replaced with `recvmmsg`/`sendmmsg` ‚Äî harvests up to N packets per syscall into a pre-allocated message vector. Syscall cost amortized across the batch                                                                                            | Full kernel network stack traversed per packet regardless ‚Äî `sk_buff` alloc, IP/UDP parse, socket layer  |
| `v6`    | `iou_mt`         | N threads each own an `io_uring` ring. Submissions queued without syscall; `SQPOLL` kernel thread drains the ring. Zero syscalls on hot path                                                                                                               | Kernel network stack still traversed; `sk_buff` still allocated per packet                               |
| `v7`    | `xdp`            | eBPF XDP program intercepts packets at the NIC driver level before `sk_buff` allocation, redirects into a UMEM region shared with userspace. One thread per NIC RX queue polls the `AF_XDP` ring directly ‚Äî kernel stack entirely bypassed, near-zero-copy | XDP-capable NIC required; eBPF program + UMEM ring management add significant setup complexity           |

## Iteration

Each version changes exactly one variable so the benchmark delta is attributable to a single cause.

- **v1 ‚Üí v2** ‚Äî add cores. `SO_REUSEPORT` lets N threads each own a socket on the same port; the kernel distributes packets across them. Removes the single-core ceiling but threads still block.
- **v2 ‚Üí v3** ‚Äî eliminate blocking. Drop back to single-threaded to isolate the effect: `epoll` + `O_NONBLOCK` means the thread never stalls inside the kernel. Removes idle CPU but reintroduces the single-core ceiling.
- **v3 ‚Üí v4** ‚Äî restore cores. Apply `SO_REUSEPORT` to the non-blocking model ‚Äî N independent `epoll` loops, one per thread. Now both cores and non-blocking are combined. Per-packet syscall cost remains.
- **v4 ‚Üí v5** ‚Äî batch syscalls. Replace `recvfrom`/`sendto` with `recvmmsg`/`sendmmsg`. One syscall handles N packets. Syscall overhead drops sharply; bottleneck shifts to the kernel network stack.
- **v5 ‚Üí v6** ‚Äî eliminate syscall transitions. `io_uring` submission/completion rings let userspace queue I/O without crossing the kernel boundary. `SQPOLL` achieves zero syscalls on the hot path. Stack traversal cost remains.
- **v6 ‚Üí v7** ‚Äî bypass the kernel stack. An eBPF XDP program intercepts packets before `sk_buff` allocation and redirects them into UMEM. `AF_XDP` rings give userspace direct access to packet data. `sk_buff`, IP/UDP parsing, and the socket layer are eliminated entirely.
