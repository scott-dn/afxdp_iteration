## Code References

When asked about a specific line number, read the file first and confirm which line is being referred to before answering. Always verify line references against the actual file content.

## Project Context

Build system uses Make. When modifying Makefiles, prefer simple explicit rules over complex pattern matching with eval. Test the build after any Makefile changes by running `make clean && make`.

## Communication Style

When asked a question or requested a fix, provide the concrete actionable fix (commands to run, files to edit) on the first response.
